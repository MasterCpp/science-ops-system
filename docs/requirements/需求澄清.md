# 需求澄清记录

## 项目定位

本项目是科普活动运营管理系统，用于管理科普活动从创建、报名、签到、志愿者管理、来访报备、满意度问卷、照片归档到数据导出的完整流程。

这个项目同时用于练习企业级项目从 0 到 1 的完整流程。当前阶段只做需求和项目文档，不写业务代码。

## 已确认需求

### 项目范围

- 第一版按完整方案 B 推进。
- 第一版是 Web 系统，包括后台管理端和手机 H5 页面。
- 第一版不做微信小程序。
- 第一版不做公众号自动发布。
- 第一版不要求绑定外包方私有服务。

### 技术方向

- 后台管理端默认采用 Vue 3 + Element Plus。
- 手机端默认采用 H5 页面。
- 后端候选为 Spring Boot 或 NestJS，实施前通过 ADR 锁定。
- 数据库默认采用 MySQL 8。
- 部署默认采用 Docker + Nginx + HTTPS。
- 活动照片第一版默认存储在服务器本地目录。

### 用户角色

- 超级管理员
- 活动管理员
- 志愿者管理员
- 超级管理员负责全局管理。
- 活动管理员负责活动主流程。
- 志愿者管理员只管理志愿者相关功能，不管理报名、问卷、来访报备和系统账号。

### 手机端访问

- 观众和志愿者第一版不需要账号登录。
- 用户通过活动链接或二维码进入手机端页面。
- 第一版默认通过手机号等报名信息识别用户。

### 活动状态

- 草稿
- 报名中
- 进行中
- 已结束
- 已归档

### 观众报名

- 观众报名提交即成功。
- 报名表采用固定基础字段 + 少量自定义字段。
- 观众报名固定字段采用基础实名方案：姓名、手机号、报名人数、单位/学校、年龄段、备注。
- 活动有报名截止时间。
- 超过报名截止时间后，手机端不允许继续报名。
- 管理员后台可以补录报名。
- 用户手机端不能取消报名。
- 管理员后台可以取消报名记录。
- 同一活动下，同一手机号只能报名一次。

### 签到

- 每场活动生成固定签到二维码。
- 用户扫码进入签到页面完成签到。
- 同一报名记录只能签到一次。
- 重复扫码时显示已签到，不新增记录。
- 活动进行中允许签到。
- 活动结束后不允许用户手机端签到。
- 管理员后台可以手动补签。
- 管理员后台可以撤销签到。
- 观众不需要签退。

### 志愿者

- 志愿者按活动岗位报名。
- 志愿者报名需要后台审核。
- 志愿者服务时长按签到和签退时间计算。
- 志愿者报名字段：姓名、手机号、单位/学校、年龄段、可服务时间说明、过往志愿服务经验、备注、选择岗位。
- 第一版不做完整志愿者档案库，按每场活动报名岗位管理。
- 每个活动可创建多个志愿者岗位。
- 岗位字段：岗位名称、岗位说明、人数上限、服务开始时间、服务结束时间。
- 志愿者只能报名同一活动下的一个岗位。
- 岗位满员后不能继续报名。
- 第一版不支持志愿者候补。
- 管理员可以审核通过或拒绝志愿者报名。
- 管理员可以手动修正志愿者服务时长。
- 志愿者审核通过后才能签到。
- 志愿者同一岗位只能签到一次、签退一次。
- 志愿者可以在活动进行中签到和签退。
- 管理员可以后台补签、补签退、撤销签到、撤销签退。
- 服务时长默认按签退时间减签到时间计算。
- 第一版不做正式志愿者服务证明模板，只导出志愿者服务数据 Excel。

### 来访报备

- 来访报备作为独立登记模块实现。
- 来访报备可以关联活动。
- 来访报备字段：来访单位、联系人姓名、联系人手机号、来访人数、来访日期、来访事由、关联活动、备注。
- 关联活动可以为空。

### 活动方案和流程

- 活动方案支持富文本说明和附件上传。
- 活动流程使用时间轴条目管理。

### 满意度问卷

- 每场活动可以配置满意度问卷。
- 问卷支持自定义题目。
- 第一版题型目标包括单选、多选、评分、文本。
- 每场活动第一版创建一份满意度问卷。
- 问卷默认实名，关联报名人手机号和姓名。
- 每个报名手机号每场活动只能提交一次问卷。
- 问卷提交后不能修改。
- 只有已签到用户才能填写问卷。
- 后台可以查看问卷统计和原始明细。
- 后台可以导出问卷结果 Excel。

### 照片归档

- 照片按活动归档。
- 第一版采用服务器本地存储。
- 活动照片支持 JPG、JPEG、PNG、WEBP。
- 单张照片最大 10MB。
- 支持后台预览照片。
- 支持单张删除。
- 第一版不做回收站恢复。
- 支持按活动批量下载照片 ZIP。

### 导出和报告

- 支持报名表、签到表、志愿者表、满意度表导出 Excel。
- 支持简单活动总结数据页。
- 第一版不做固定单位 Logo、表头或盖章位置模板。
- 报名表导出列：活动名称、姓名、手机号、报名人数、单位/学校、年龄段、备注、报名时间、报名状态。
- 签到表导出列：活动名称、姓名、手机号、签到时间、签到方式、是否补签。
- 志愿者表导出列：活动名称、岗位、姓名、手机号、审核状态、签到时间、签退时间、服务时长、是否人工修正。
- 问卷结果导出列：活动名称、姓名、手机号、提交时间、题目、答案。
- 来访报备导出列：来访单位、联系人、手机号、来访人数、来访日期、来访事由、关联活动、备注。
- 活动总结数据页指标：报名人数、实际签到人数、签到率、志愿者报名人数、志愿者通过人数、志愿者总服务时长、问卷提交人数、平均满意度评分、照片数量。

### 数据安全

- 后台登录需要账号密码。
- 密码加密存储。
- 后台操作记录操作日志。
- 操作日志记录范围：登录、创建/修改/删除活动、发布/结束/归档活动、导出数据、补录/取消报名、补签/撤销签到、审核志愿者、删除照片/附件。
- 手机号默认完整显示给有权限的后台角色。
- 第一版不做手机号脱敏。
- 第一版提供数据库备份说明文档，不做自动备份功能。

### 活动发布后修改规则

- 草稿状态：所有字段可修改。
- 报名中：基础信息可修改，但报名字段、人数上限、报名截止时间修改要记录日志。
- 进行中：不允许修改报名字段和人数上限，只允许修改方案、流程、附件、照片。
- 已结束：只允许补充照片、附件、总结数据。
- 已归档：默认只读，超级管理员可解除归档后修改。

- 观众报名固定字段已确认：姓名、手机号、报名人数、单位/学校、年龄段、备注。

### 活动规则

- 活动支持设置报名人数上限，未设置上限时视为不限人数。
- 达到报名人数上限后停止报名。
- 第一版不支持候补名单。
- 活动有报名截止时间。
- 报名截止后管理员可以后台补录。
- 活动发布后修改规则已确认，详见上方“活动发布后修改规则”。

### 签到规则

- 同一报名记录只能签到一次，重复扫码显示已签到。
- 活动进行中允许签到。
- 活动结束后不允许用户手机端签到。
- 管理员后台可以手动补签。
- 管理员后台可以撤销签到。
- 观众不需要签退。

### 志愿者规则

- 岗位支持人数上限。

- 第一版不支持志愿者候补。
- 服务时长允许管理员人工修正。
- 志愿者审核通过后才能签到。
- 志愿者同一岗位只能签到一次、签退一次。
- 志愿者可以在活动进行中签到和签退。
- 管理员可以后台补签、补签退、撤销签到、撤销签退。
- 服务时长默认按签退时间减签到时间计算。
- 第一版不做正式志愿者服务证明模板，只导出志愿者服务数据 Excel。

### 问卷规则

- 问卷默认实名，关联报名人手机号和姓名。
- 每个报名手机号每场活动只能提交一次。
- 提交后不允许修改。
- 只有已签到用户才能填写问卷。

### 文件和照片

- 活动方案附件支持 PDF、Word、Excel、PPT、图片。
- 单个附件最大 20MB。
- 活动照片支持 JPG、JPEG、PNG、WEBP。
- 单张照片最大 10MB。
